# 烟火小食单 Batch1 管理后台 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 `menu-admin` 仓库内从零搭建 Vue3 管理后台，完成 Batch1 的登录、路由守卫、权限映射、主数据管理（菜品/食材）和账号角色权限配置页面。

**Architecture:** 使用 Vite + Vue3 + TypeScript 建立单页后台，`src/api` 负责接口调用，`src/stores` 负责登录态与权限态，`src/views` 按业务模块拆页面，页面层统一采用 A 版“业务优先”侧栏结构。权限以 API 权限点为真源，前端只负责 route meta 和按钮级展示控制。

**Tech Stack:** Vue 3, Vite, TypeScript, Vue Router, Pinia, Axios, Element Plus, Vitest

```ts
export const useAuthStore = defineStore('auth', {
  state: () => ({
    accessToken: '',
    refreshToken: '',
    permissions: [] as string[],
    clientScopes: [] as string[],
  }),
  actions: {
    setSession(payload: LoginSession) {
      this.accessToken = payload.accessToken
      this.refreshToken = payload.refreshToken
      this.permissions = payload.permissions
      this.clientScopes = payload.clientScopes
    },
  },
})
```
